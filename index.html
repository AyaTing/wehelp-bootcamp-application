關於這份申請網頁，分享一個開發時的技術心得。 +

實際親手製作自己的網頁後，發現許多在學習或是練習課程範例時沒思考過的問題， 例如：在製作JS30的展示網頁時，本來打算用CSS中的scroll-timeline和view-timeline來實現立體卡片的輪播效果，結果完成之後才突然發現這些屬性在safari上不支援，只好重新用JavaScript的getBoundingClientRect() 來計算卡片相對位置，才勉強做出接近的效果。
這次在著手製作申請書前，已經先決定好要嘗試什麼網頁效果，並在實作中邊在MDN查詢不熟悉的屬性是否有被廣泛支援，正當以為全部都完成時，結果看到在safari上手風琴的標題的hover效果看起來很像破圖🤦‍♀️
再重新查看MDN才看到clip-path雖然能在safari上使用，但不支援clip-path的動畫屬性（Is animatable這欄safari是❌）。
於是想到CSS能用@supports來寫判斷規則，檢查瀏覽器是否支援特定屬性，並提供替代方案，然而因為safari其實是支援clip-path的，所以試了好幾種寫法都無法讓規則只排除掉safari，無計可施之下只好求助ChatGPT，最後使用 JavaScript 來檢查瀏覽器是否為 Safari，並根據檢查結果添加特定的 CSS 類別來處理。
最終雖然clip-path沒辦法在safari上看到動畫過渡的狀態，但仍保留Hover效果，讓我蠻意外的。 總之透過這些製作的過程讓我實際了解到網頁製作有許多必須留意的細節，一個網頁效果可能有好幾種不同的程式寫法，我想也是寫程式的魅力之處吧。
